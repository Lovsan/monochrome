Navigation

Represents a collection of messages that can be navigated in two dimensions via reaction buttons below the message. A navigation is a collection of NavigationChapters. Each navigation chapter controls a one-dimensional slice of navigable content. The user can move forward and backward within a chapter by using the arrow reaction buttons, and can switch chapters by using the other reaction buttons.

Once a Navigation is constructed, it can be shown to the user by using NavigationManager#show. For an example of creating a navigation, see the demo navigation command.

Constructor

new Navigation(ownerId, showPageArrows, initialEmojiName, chapterForEmojiName)

Source:
  • navigation.js, line 43
Construct a Navigation. If this Navigation only has one NavigationChapter, use the Navigation.fromOneNavigationChapter factory method instead. If you only need left and right pagination, and you already have all the content you want to display to the user, use the Navigation.fromOneDimensionalContents factory method instead.
Parameters:
Name Type Description
ownerId string The user ID of the user who is allowed to use the reaction buttons to navigate.
showPageArrows boolean Whether to show the arrow reactions or not. If you know in advance that none of your NavigationChapters have more than one page, you can set this to false.
initialEmojiName string The name of the emoji for the chapter to show initially.
chapterForEmojiName Object.<string, NavigationChapter> An object where the keys are emoji names and the values are NavigationChapters. When the emoji is clicked, control will switch to the specified chapter. To find the name of an emoji, enter the emoji in Discord's chat box, put a \ in front of it, and hit enter.

Construct a Navigation with just one chapter. Only the arrow reactions will be shown and navigation will only be possible in one dimension. If you already have all the content you want to display to the user, you can use the Navigation.fromOneDimensionalContents factory method instead and avoid having to construct a NavigationChapter yourself.
